# Машиночитаемые датасеты Красной книги Российской Федерации (Флора и Фауна)

## Описание проекта
Настоящий репозиторий содержит первые унифицированные, структурированные, машиночитаемые наборы данных по редким и исчезающим видам животных, растений, грибов и лишайников, созданные на основе официальных актуальных изданий Красной книги Российской Федерации.

Проект разработан в рамках концепции открытой науки (**Open Science**) и методологии **FAIR** (Findable, Accessible, Interoperable, Reusable — Находимость, Доступность, Совместимость, Повторное использование). Основная цель проекта — извлечь данные из труднообрабатываемых монолитных PDF/HTML томов и перевести их в структурированный вид, пригодный для автоматизированного статистического анализа, ГИС-моделирования, интеграции в веб-приложения и построения прогностических моделей машинного обучения.

## Спецификация метаданных (Схема данных)
Оба результирующих датасета (Флора и Фауна) полностью приведены к единой структуре данных, состоящей из 14 нормализованных атрибутов (столбцов):

```text
| № | Атрибут (ID поля) | Тип данных | Описание поля |
|---| :--- |---| :--- |
| 1 | `latin_name` | String | Валидированное научное латинское название таксона (Первичный ключ / Primary Key) |
| 2 | `page` | Integer | Номер первой страницы очерка в официальном печатном томе Красной книги |
| 3 | `russian_name`| String | Официальное русскоязычное название вида |
| 4 | `family` | String | Биологическое семейство таксона |
| 5 | `genus` | String | Биологический род (выделен программным путем из латинского наименования) |
| 6 | `category_numeric`| Integer | Цифровой код категории статуса редкости вида (от 0 до 5 согласно Приказам Минприроды) |
| 7 | `category_text`| String | Буквенные коды природоохранного статуса (например: У, КР, III, БУ) |
| 8 | `distribution` | String | Нормализованное текстовое описание географического ареала распространения вида |
| 9 | `habitat` | String | Описание экологических ниш, биотопов и специфики биологии/экологии вида |
| 10| `abundance` | String | Данные о численности популяций и текущей популяционной динамике |
| 11| `limiting_factors`| String | Естественные и антропогенные лимитирующие факторы, угрожающие таксону |
| 12| `protection_measures`| String | Описание уже принятых государственных, заповедных и региональных мер охраны |
| 13| `additional_measures`| String | Рекомендации научного сообщества по необходимым дополнительным мерам сохранения |
| 14| `author` | String | Верифицированные ФИО авторов-составителей очерка, нормализованные по маске инициалов |
```

## Особенности реализации конвейера (Pipeline)
1. **Сверхточная сегментация (Сutting):** В отличие от стандартной постраничной нарезки, алгоритм анализирует координатные теги `top` в HTML-верстке. При обнаружении маркера нового очерка (классы `ft01`/`ft028`) страница динамически разрезается попиксельно. Это предотвращает попадание «хвостов» предыдущих видов в файлы новых таксонов.
2. **Двухколоночный координатный сбор данных:** Поскольку исходный текст сверстан в две колонки, стандартное чтение строк ломает логику текста. Парсер группирует элементы по координате `left` (левая колонка `< 430px`, правая `≥ 430px`) и сортирует их по вертикальной оси `top`, обеспечивая точную семантическую сборку.
3. **Автоматическая постобработка (Data Cleansing):** - Склейка мягких переносов строк и удаление служебных артефактов верстки.
   - Срезание избыточных заголовков внутри ячеек (например, регулярные выражения удаляют фразы «и особенности экологии» в начале текстовых блоков).
   - Выделение авторов строго по регулярной маске `[А-Я]\.[А-Я]\.\s*[А-Я][а-я]+` (Инициалы Фамилия) с автоматической дедупликацией авторов на лету.

## Технологический стек и воспроизводимость
- **Язык разработки:** Python 3.9+
- **Ключевые библиотеки:** `pandas`, `beautifulsoup4`, `lxml`, `pathlib`
- **Промежуточный инструмент конвертации:** CLI-утилиты пакета `poppler-utils` (`pdftohtml`).

Процесс сборки датасетов полностью детерминирован. Для воспроизведения конвейера необходимо настроить пути к локальным директориям в скриптах и последовательно запустить их из папки `scripts/`.
